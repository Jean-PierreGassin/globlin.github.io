World of Warcraft Auction Intelligence

Globlin AH Bot

A Discord bot that scans live auction house data and posts alerts when tracked items are listed far below market value.

Quick Start Deploy This Page

Core Features

Real-Time Auction Scans

Continuously checks Blizzard auction data and flags listings at or below your configured undercut threshold.

Per-Server Tracking

Each Discord guild has isolated tracked items and subscription status for true multi-tenant operation.

Item-Level Aware Pricing

Supports item-level-specific tracking for gear where value shifts heavily by ilvl.

Stripe Billing Support

Optional Stripe event polling enables subscription activation and status updates without webhooks.

Web Control Panel

Optional local panel to start or stop the bot process, inspect logs, and review tracked item summaries.

Quick Start

  1. Clone the bot repository

    git clone git@github.com:Jean-PierreGassin/auction-house-scraper.git
cd auction-house-scraper

  2. Create and activate a virtual environment

    python -m venv .venv
source .venv/bin/activate

  3. Install dependencies

    pip install -r requirements.txt

  4. Create local config and subscription files

    cp config.json.example config.json
cp customer-subscriptions.json.example customer-subscriptions.json
mkdir -p customer-tracked-items

  5. Fill required credentials in config.json

    {
  "discord": { "token": "YOUR_DISCORD_TOKEN" },
  "blizzard": {
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET"
  },
  "access": { "owner_user_ids": [123456789012345678] }
}

  6. Run the bot

    python main.py

    Optional: run the web control panel with python web_server.py.

Discord Commands

/setup Post onboarding and subscribe button.

/subscription_status Show guild subscription state.

/items Open guided tracked-item management panel.

/upsert_subscription Owner-only subscription write.

/remove_subscription Owner-only subscription remove.

/list_subscriptions Owner-only subscription list.

How Alerts Work

  • Bot loads cached item + connected realm data at startup.
  • It polls auction houses on a recurring async loop.
  • Listings are compared to market averages or configured item-level price targets.
  • Matches trigger formatted Discord alerts in each subscribed guild's configured channel.
